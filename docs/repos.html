# Repositories

These are all the repositories that Cloud CNC is comprised of. The use of NodeJSopen in new window is implied and required for most repositories.

# Frontendopen in new window

# Coreopen in new window

Koaopen in new window + SocketIOopen in new window + TypeScriptopen in new window REST and websocket API server. Uses Mongoopen in new window for persisted storage and optionally Redisopen in new window for high availability (Via the Redis SocketIO adapteropen in new window). Responsible for managing all resources, authentication (Of both clients and controllers), and connecting the frontend and controller via a SocketIO roomopen in new window.

# Controlleropen in new window

SocketIOopen in new window + TypeScriptopen in new window websocket client. Connects to the core (To eliminate additional networking configuration).

Threadsopen in new window + TypeScriptopen in new window wrapper around Cloud CNC's Cura Engine fork. Uses Unified 3D Loader for mesh preprocessing/broader file support and Threads Rollup Plugin to enable importing NPM packages in worker threads.

# Cura Engineopen in new window

Wasmeropen in new window port of Cura Engineopen in new window. Designed to stay close to the upstream source while still compiling to WASM.

# Setup Wasmeropen in new window

# WAPM Publishopen in new window

# Vue 3D Vieweropen in new window

# Unified 3D Loaderopen in new window

# Threads Rollup Pluginopen in new window

# Threads Webpack Pluginopen in new window
