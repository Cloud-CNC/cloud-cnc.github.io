# Introduction

# Architecture

Cloud CNC consists of 3 different components

  • Frontend
  • Core
  • Controller

# Frontend

The frontend uses the Vue UI framework based and the Vuetify UI system. It uses Vue CLI to manage builds. The frontend connects to the core via the HTTP API.

The repository is located at github.com/cloud-cnc/frontend.

# Core

The core glues the frontend and controller(s) together. It uses Express for HTTP routing and Mongo for a database. The core is both a raw HTTP and a websocket server.

The repository is located at github.com/cloud-cnc/core.

TIP

The core does need to be port forwarded in order for the frontend to make HTTP requests to it and for the controller(s) to initiate web socket sessions.

# Controller

The controller is meant to run on a small computer such as a Raspberry Pi and directly connects to your CNC machines typically over a USB or serial connection. The controller connects to the core via a Websocket API.

The repository is located at github.com/cloud-cnc/controller.

TIP

Because the controller(s) connect to the core, they typically only need an internet connection, you don't need to port forward them.
